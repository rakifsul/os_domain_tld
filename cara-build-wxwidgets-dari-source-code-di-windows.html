Cara Build wxWidgets dari Source Code di Windows

RAKIFSUL December 23, 2023 Belajar Build

Apa itu Windows?

Windows adalah sistem operasi yang dikembangkan oleh Microsoft.

Apa itu wxWidgets?

wxWidgets adalah sebuah library untuk bahasa pemrograman C++ untuk pembuatan GUI native yang bersifat gratis dan open source.

Yang dimaksud dengan native dalam konteks ini adalah bahwa tampilan GUI wxWidgets selaras dengan GUI yang disediakan oleh sistem operasinya.

Apa itu build?

Build, dalam hal ini, artinya adalah membangun suatu library siap pakai (dalam hal ini siap di-link) dari source code nya. (*1)

Ini lebih mudah dipahami jika Anda pernah menggunakan C++.

Di C++, suatu dependency bisa langsung berbentuk source code atau library.

Jika dalam bentuk source code, proses build dependency bisa dilakukan bersama-sama dengan source code program utama.

Jika dalam bentuk library, proses build dependency dilakukan sebelum source code program utama di-compile. Setelah source code program utama di-compile, maka dependency tadi akan di-link, kemudian keseluruhan program selesai di-build.

Catatan (*1):

Walaupun begitu, ini adalah pengertian proses build jika yang di-build adalah library.

Jika yang di-build adalah aplikasi, maka maknanya adalah proses membangun suatu executable dari aplikasi tersebut, yang di Windows format-nya adalah .exe.

Kenapa kita perlu mem-build wxWidgets?

Kita perlu melakukannya jika library siap link dari wxWidgets tidak tersedia untuk IDE (Integrated Development Environment) C++ yang kita gunakan.

Bahkan IDE yang sama tapi beda versinya terkadang library-nya tidak kompatibel satu sama lain.

Oleh sebab itu, dalam beberapa kasus kita perlu mem-build wxWidgets dari source code nya.

Apa saja yang saya perlukan untuk mem-build wxWidgets?

Anda akan memerlukan Visual Studio Community 2022 dengan komponen Desktop Development with C++.

Link download nya ada di:

https://visualstudio.microsoft.com/vs/community/

Untuk komponen Desktop Development with C++ itu sendiri merupakan pilihan dalam proses install Visual Studio Community 2022. Jadi, pastikan pada saat meng-install Visual Studio Community 2022, Anda mencentang-nya.

Selain itu, Anda juga memerlukan source code wxWidgets versi 3.0.5.

Link download-nya ada di:

https://github.com/wxWidgets/wxWidgets/releases/download/v3.0.5/wxWidgets-3.0.5.zip

OK. Saya sudah menyiapkannya. Lalu, bagaimana cara mem-build-nya?

Langkah pertama, ekstrak file zip dari wxWidgets tadi.

Anda bisa menggunakan menu Windows Explorer, atau dengan aplikasi 7zip.

Jika menggunakan 7zip, caranya adalah dengan memilih “Extract to wxWidgets-3.0.5“ seperti di gambar ini.

Setelah peng-extract-an selesai, buka “Windows Explorer“ dan buka folder “wxWidgets-3.0.5“.

Masuklah ke dalam folder “build/msw“ dan cari file “wx_vc12.sln“.

Buka file .sln tersebut dengan IDE “Visual Studio Community 2022“.

Nanti file tadi akan dikonversi ke versi yang sesuai dengan Visual Studio Community 2022. Jika Visual Studio Community 2022 menawarkan “Retarget Projects“ di situ, klik ok saja.

Lanjut, buka “menu > Build > Batch Build“.

Dialog yang berisi daftar centang akan terbuka

Di situ, klik “Select All“ saja.

Lanjut, klik “Build“.

Prosesnya akan cukup lama. Harap sabar menunggu.

OK. Proses build sudah selesai. Apa yang sebaiknya saya lakukan selanjutnya?

Saat proses build sudah selesai, akan ada banyak file .lib yang dihasilkan.

File-file .lib nya berada di folder lib.

Lebih detail:

  • Library .dll dan .lib diletakkan di “lib/vc_dll“
  • Library static diletakkan di “lib/vc_lib“
  • Library versi x64 dari kedua jenis library tadi diletakkan di “vc_x64_dll“ dan “vc_x64_lib“

Mungkin Anda akan bertanya: file library mana yang saya perlu gunakan?

Untuk mengetahui library mana yang dibutuhkan, akan lebih baik jika Anda mengecek contoh kode yang telah disediakan wxWidgets.

Ke depannya, Anda tinggal melihat file .lib apa yang digunakan dengan melihat project setting dari contoh-contoh kode yang disediakan wxWidgets.

Untuk mencoba contoh kode yang disediakan wxWidgets, “Save All“ terlebih dahulu project saat ini, kemudian close Visual Studio Community 2022.

Selanjutnya, masuk ke folder “samples/aui“.

Buka file “auidemo_vc9.vcproj“ dengan IDE Visual Studio Community 2022.

Jika ada tawaran “One Way Upgrade“, klik OK.

Nanti, format project tersebut akan dikonversi ke format project yang kompatibel dengan Visual Studio Community 2022.

Saat file project baru dibuka, maka “Save All“, dan jika ada tawaran untuk membuat solution, save dengan nama tertentu (bebas).

Lanjut, buka “menu > Debug > Start Without Debugging“.

Jika proses build berhasil, akan tampil sebuah window GUI seperti gambar ini.

OK. Sekarang saya mengerti. Terima kasih ya…

Sama-sama.
